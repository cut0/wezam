# AI リソース管理ガイド

CLAUDE.md やルールファイルの作成・更新ルール。

## 更新タイミング

コード変更と AI リソースの更新はセットで行う。

| 変更内容                     | 更新対象                           |
| ---------------------------- | ---------------------------------- |
| ワークフローの追加・変更     | 対応するルールファイル             |
| 新しいワークスペースの追加   | アーキテクチャルール + `CLAUDE.md` |
| 開発コマンドの追加・変更     | `CLAUDE.md`                        |
| パッケージ名・スコープの変更 | `CLAUDE.md`                        |
| コーディング規約の変更       | 対応するルールファイル             |
| 新しい技術スタックの導入     | 新規ルールファイル + `CLAUDE.md`   |

## ルールファイルのフォーマット

### 本文の構成

```markdown
# {ルール名}ガイド

対象領域の 1 行説明。

## セクション

（ルール、パターン、コード例など）
```

- H1 は 1 つだけ
- 簡潔さを保つ（目安: 2-8KB）
- コード例は ✅ OK / ❌ NG の対比で示す
- テーブルで選択基準や対応表を整理する

## プロジェクト非依存の原則

ルールファイルには以下を記載しない:

- プロジェクト名・プロダクト名
- パッケージスコープ名（`@scope/pkg`）
- 具体的な URL・エンドポイント
- 機能名・画面名

これらのプロジェクト固有情報は `CLAUDE.md` に集約する。ルールは他プロジェクトにコピーして転用可能な抽象度を保つ。

## CLAUDE.md の更新ルール

`CLAUDE.md` はプロジェクトのエントリポイント。以下のセクションを管理する。

| セクション           | 内容                           |
| -------------------- | ------------------------------ |
| プロジェクト概要     | プロダクトの 1 行説明          |
| 開発コマンド         | スクリプト一覧                 |
| ワークスペース構成   | ディレクトリツリー             |
| プロジェクト固有情報 | パッケージ名、インポートパス等 |
